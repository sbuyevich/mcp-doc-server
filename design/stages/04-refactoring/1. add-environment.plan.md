# Add Environment-Aware NuGet Retrieval

## Summary

Persist an environment separately from the NuGet source and expose environment-qualified library IDs such as `nuget:qa/Company.Package`. Agents can select an exact environment and version, while legacy `nuget:Company.Package` requests remain supported through configured precedence.

## Key Changes

### Configuration and persistence

- Add required `Environment` to each Indexer CLI `NuGetSourceOptions`; validate it as a case-insensitive slug using letters, numbers, `.`, `_`, and `-`.
- Allow multiple sources to share an environment.
- Add `Retrieval.EnvironmentOrder`; retain `SourceOrder` for feed precedence within an environment.
- Add schema migration v3 with `sources.environment`.
- Populate migrated rows with their source name, then update them from Indexer CLI configuration during indexing.
- Preserve existing source, library, version IDs, and `nuget://` citation URIs.

### MCP contracts and identifiers

- Support:
  - Qualified: `nuget:qa/Company.Package`
  - Legacy: `nuget:Company.Package`
- Add optional `Environment` to `ResolveLibraryRequest`.
- Make `resolve_library` return one match per package and environment using qualified IDs.
- Add `Environment` and selected `SourceId` to each library match and resolved context.
- Keep the four existing tools and resource templates unchanged.

### Selection behavior

- Qualified IDs restrict retrieval to the named environment.
- Legacy IDs select by `EnvironmentOrder`, then `SourceOrder`.
- Within an environment, select the first configured source containing a usable version; unlisted entries sort after configured entries by ordinal name.
- `resolve_library(Environment: "qa")` searches only QA.
- Return explicit `environment_not_found` or `library_not_found` errors rather than falling back to another environment.
- Recommended-version precedence:
  1. Qualified key such as `nuget:qa/Company.Package`
  2. Existing package-wide key such as `Company.Package`
  3. Normal stable-version fallback
- Exact request `Version` continues to take precedence over recommendations.

## Test Plan

- Test qualified and legacy library ID parsing, formatting, invalid slugs, and case-insensitive environments.
- Test configuration validation for required environments, duplicates in ordering lists, and multiple sources sharing one environment.
- Test schema v2-to-v3 migration and environment updates without changing stable package IDs.
- Index the same package in QA and production with different versions and verify:
  - discovery returns separate qualified matches;
  - environment filtering works;
  - qualified requests never cross environments;
  - legacy requests follow both precedence lists;
  - exact versions and environment-specific recommendations work;
  - responses report environment and concrete source.
- Test multiple feeds in one environment and source tie-breaking.
- Update serialization, MCP schema, stdio, HTTP, Indexer, architecture, and documentation tests.
- Run the complete build, test suite, and stale-contract searches.

## Assumptions

- Environment is selection metadata, not an authorization boundary.
- Source name remains the concrete feed identity used by citations.
- Indexer CLI configuration must explicitly declare every source environment.
- Deploy and run the upgraded Indexer CLI before starting the upgraded Host so schema v3 exists.
- Renaming an environment intentionally changes its public qualified library IDs.
